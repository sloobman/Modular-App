# Modular App

Простой модульный консольный .NET-проект, демонстрирующий динамическую загрузку и инициализацию модулей.

## Краткое описание

Проект разделён на две части:
- `src/Core` — инфраструктура загрузки, разрешения зависимостей и запуска модулей.
- `src/Modules` — пример модулей приложения: `Logging`, `Data`, `Validation`, `Report`, `Export`.

Цель проекта — показать архитектуру, в которой новое поведение добавляется как отдельный модуль без изменения логики запуска.

## Структура проекта

- `src/Program.cs` — точка входа приложения. Загружает модули, разрешает их зависимости и запускает.
- `src/Core/IModule.cs` — интерфейс, который должен реализовать каждый модуль.
- `src/Core/ModuleLoader.cs` — читает список модулей из `appsettings.json`, загружает модули из каталога `modules` и из текущей сборки.
- `src/Core/DependencyResolver.cs` — упорядочивает модули в правильном порядке по зависимостям.
- `src/Core/ModuleRunner.cs` — регистрирует сервисы модулей и запускает их инициализацию.
- `src/Modules/*.cs` — примеры модулей, которые реализуют `IModule`.

Новое поведение:
- `DataModule` — предоставляет данные через `IDataProvider`.
- `ValidationModule` — проверяет данные и логирует проблемы.
- `ReportModule` — формирует отчет по данным.
- `ExportModule` — экспортирует данные в CSV-файл.

## Как запустить

Откройте терминал в корне репозитория `Modular-App` и выполните:

```powershell
dotnet build ModularApp.sln
```

Для запуска приложения:

```powershell
dotnet run --project src/ModularApp.csproj
```

Для выполнения тестов:

```powershell
dotnet test ModularApp.Tests/ModularApp.Tests.csproj
```

## Как это работает

1. `Program.cs` читает список модулей из `appsettings.json`.
2. `ModuleLoader` загружает модули из каталога `modules` и из текущей сборки.
3. `DependencyResolver` выполняет топологическую сортировку модулей по их декларативным зависимостям (`Dependencies`).
4. `ModuleRunner` регистрирует сервисы модулей в `ServiceCollection`, строит `IServiceProvider` и вызывает `Initialize()` для каждого модуля.

## Что такое ядро и что такое модули

- Ядро проекта находится в `src/Core`:
  - загрузка модулей
  - разрешение зависимостей
  - запуск модулей
- Модули приложения находятся в `src/Modules`:
  - `LoggingModule` — регистрирует инициализацию логгера
- `DataModule` — предоставляет набор данных
- `ValidationModule` — зависит от `Logging` и `Data`, проверяет записи
- `ReportModule` — зависит от `Validation` и `Data`, формирует итоговый отчет
- `ExportModule` — зависит от `Logging` и `Data`, экспортирует данные в CSV
## Почему добавление нового модуля не требует редактирования логики запуска

Потому что `Program.cs` и инфраструктура модулями не жёстко привязаны к конкретным классам. Новая функциональность добавляется так:

1. создать новый класс, реализующий `IModule`;
2. объявить имя и зависимости;
3. реализовать `RegisterServices()` и `Initialize()`.

`ModuleLoader` найдёт новый класс автоматически, `DependencyResolver` учтёт зависимости, а `ModuleRunner` выполнит его вместе с остальными. Таким образом, запуск и управление модулями остаются неизменными.
